## 1. Overview

This document is a **modern-first** guide to binary exploitation and reverse engineering. It assumes:

* 64-bit systems by default
* Modern mitigations are always enabled
* Multiple architectures matter (x64, ARM64, RISC-V)
* Automation, fuzzing, and tooling are core skills

It is designed for learners moving beyond legacy x86-era material into real-world exploitation in the 2020s.

## 2. Core Books & Reading Materials

### 2.1 Essential Modern Texts

#### *Practical Binary Analysis* — Dennis Andriesse

* Focus: x64 Linux, modern toolchains
* Topics:

  * Disassembly and IRs
  * Data-flow analysis
  * Binary instrumentation (PIN)
  * Automated vulnerability discovery
* Why it matters: 64-bit-first, automation-oriented

#### *Mastering Malware Analysis* — Kleymenov & Thabet

* Platforms: Windows 10/11, modern Linux
* Topics:

  * PE/ELF internals
  * x64dbg, Ghidra, IDA workflows
  * Anti-debugging and packers

#### *The Ghidra Book* — Eagle & Nance

* Definitive guide to Ghidra
* Scripting, collaboration, processor modules

### 2.2 Architecture-Specific Reading

#### ARM / AArch64

* *Android Security Internals* — Nikolay Elenkov
* *ARM Assembly Language Programming & Architecture* — Mazidi

#### RISC-V

* *The RISC-V Reader* — Patterson & Waterman

### 2.3 Modern Toolchain & Automation

* *Black Hat Go* — Offensive tooling in Go
* *The Fuzzing Book* (free): [https://www.fuzzingbook.org/](https://www.fuzzingbook.org/)

## 11. Critical Differences from the Legacy x86 Era

### 11.1 Then vs Now

**2000s**

* Stack overflow
* Shellcode
* RET overwrite

**2020s**

* Fuzzing
* Primitive analysis
* Mitigation bypass chains
* Multi-stage exploits

### 11.2 Modern Mitigations

* ASLR, NX, RELRO, PIE
* CET, CFI, XFG
* PAC, BTI, MTE
* SMAP / SMEP / KPTI
